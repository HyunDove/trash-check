## 1. 프로젝트 개요

### 1.1 추진배경 및 목적

- 분리배출은 재질(플라스틱·캔·유리·종이) 판별이 선행되어야 하지만, 일반 시민이 혼합 재질·오염된 쓰레기의 배출 방법을 정확히 알기 어렵다.
- 이미지 분류 CNN만으로는 "무엇인지"는 알 수 있어도 "어떻게 버려야 하는지"는 답할 수 없다 — 판별(비전 모델)과 안내(LLM + 근거 문서)를 결합한 파이프라인이 필요하다.
- 사진 업로드 한 번으로 **재질 판별 → 알맞은 분리수거함 안내 → 배출 방법 질의응답**까지 이어지는 엔드투엔드 데모 앱을 구축한다.

### 1.2 과제 범위

| 과제구분 | 내용 |
|---|---|
| **AI** | **TrashNet 데이터셋 수집 및 5클래스 재구성** |
| | YOLO 게이트 구성 (COCO 사전학습 YOLOv8n — 물체 crop) |
| | CNN 전이학습 (EfficientNet-B0, 5클래스 분류) |
| | VLM 최종 판정 (재질+이물질, 로컬 Ollama / 배포 HF API 이중화) |
| | LangChain RAG 챗봇 (환경부 가이드라인 + 생활법령 문서 기반) |
| | Streamlit 시연 웹앱 구축 및 Streamlit Cloud 배포 |

### 1.3 과제 추진 방법

#### 1) 설계 원칙

- 5일 일정 내 완주를 위해 **라벨링 작업이 필요 없는 구성**을 선택한다 — YOLO는 파인튜닝 없이 COCO 사전학습 그대로, CNN은 폴더 분류 데이터셋(TrashNet)으로 전이학습.
- 각 단계는 실패해도 다음 단계로 안전하게 폴백한다 (YOLO 미감지 → 전체 이미지, VLM 실패 → CNN 판정, LLM 실패 → 안내 메시지).
- 로컬 데모(Ollama)와 클라우드 배포(HF Inference API)를 환경변수 하나(`LLM_BACKEND`)로 전환하는 이중화 구조를 적용한다.

#### 2) AI 모델 적용 대상

| 구분 | 입력 | 모델 | 목표 |
|---|---|---|---|
| 물체 탐지 게이트 | 원본 사진 (RGB) | YOLOv8n (COCO 사전학습) | 가장 큰 물체 1개 crop |
| 재질 분류 | crop 이미지 (224×224) | EfficientNet-B0 전이학습 | 5클래스 분류 (plastic/can/glass/paper/trash) |
| 최종 판정 | crop 이미지 | VLM (Ollama gemma4 / HF 비전 모델) | 재질 보정 + 이물질 검출 |
| 배출 안내 | 재질 + 질문 | LangChain RAG + Qwen2.5 7B | 근거 문서 기반 답변 생성 |

#### 3) AI 분석모델 구축 프로세스

```
[데이터 구축]      →  [CNN 학습]        →  [파이프라인 통합]     →  [평가]        →  [시연·배포]
TrashNet 다운로드     EfficientNet-B0      YOLO 게이트            검증 정확도      Streamlit 앱
6→5클래스 매핑        전이학습(Colab T4)   + CNN + VLM 판정       혼동행렬         Streamlit Cloud
2,527장               증강·클래스가중치    + RAG 챗봇             클래스별 지표    로컬/배포 이중화
```

### 가. CNN 학습 데이터 — TrashNet 재구성

Stanford CS229의 공개 데이터셋 **TrashNet**(Gary Thung & Mindy Yang)을 내려받아, 원본 6클래스를 과제 5클래스로 매핑해 구축했다 (`scripts/download_dataset.py`).

| 클래스 | 장수 | 원본 TrashNet 클래스 | 매핑 근거 |
|---|---:|---|---|
| paper (종이) | 997 | paper + cardboard | 국내 분리배출 기준상 종이류로 통합 배출 |
| glass (유리) | 501 | glass | 그대로 사용 |
| plastic (플라스틱) | 482 | plastic | 그대로 사용 |
| can (캔) | 410 | metal | 생활 쓰레기에서 금속류 대부분이 캔 |
| trash (일반쓰레기) | 137 | trash | 이물질·판별불가 항목의 목적지 클래스 |
| **총합** | **2,527** | | |

- 학습/검증 분할: **8:2** (seed 42 고정)
- **클래스 불균형 대응**: paper(997)가 trash(137)의 약 7배 → 학습 시 클래스별 가중치 `total / (num_classes × count)`를 CrossEntropyLoss에 적용

## 3. 시스템 아키텍처 — 4단 판별 파이프라인

### 가. 전체 파이프라인

```
📷 사진 업로드 (쓰레기통 투입구 UI — 클릭 또는 드래그드롭)
     ↓
[1] 🔍 YOLO 게이트 (model/detect.py)
    COCO 사전학습 YOLOv8n (conf=0.25, 파인튜닝 없음)
    ├─ 0개 감지 → 전체 이미지 사용 (COCO 미포함 물체 폴백)
    └─ 1개+ 감지 → 바운딩 박스 면적 최대 물체만 crop
     ↓
[2] 🧠 CNN 분류 (model/predict.py)
    EfficientNet-B0 전이학습 → 5클래스 확률 (CPU 추론)
     ↓
[3] 👁️ VLM 최종 판정 (model/vision.py)
    재질 + 이물질을 한 번에 판단 (로컬 Ollama / 배포 HF API)
    사용 가능 → VLM 결과 최종 채택 / 실패 → CNN + 확신도 70% 폴백
     ↓
[4] 📚 RAG 챗봇 (rag/chain.py)
    판별된 재질 + 사용자 질문 → Chroma 벡터DB 검색(k=8) → 근거 기반 답변
     ↓
🗑️ 분리수거함 투입 애니메이션 + 챗봇 패널 (Streamlit)
```

### 나. 단계별 폴백 설계

모든 단계가 실패해도 앱이 죽지 않도록, 각 단계에 안전한 폴백 경로를 두었다.

| 단계 | 실패 상황 | 폴백 동작 |
|---|---|---|
| YOLO 게이트 | 물체 0개 감지 (COCO 미포함) | 전체 이미지를 그대로 CNN 입력으로 사용 |
| YOLO 게이트 | crop 좌표 계산 예외 | 원본 이미지로 진행 |
| VLM 판정 | 토큰 없음·모델 미지원·크레딧 소진 | CNN 분류 + 확신도 임계값(70%)으로 판정 |
| RAG 챗봇 | LLM 연결 실패·API 장애 | `safe_ask()`가 트레이스백 대신 안내 메시지 표시 |

### 다. 최종 판정 로직 (`judge()`)

```python
if vlm_result:                      # VLM 사용 가능 → VLM 판단 최종 채택
    if vlm_result["contaminated"]:  # 이물질 감지
        return "trash", "이물질이 감지됐어요 — 깨끗이 씻으면 재활용 가능"
    return vlm_result["material"], "재질 판별 완료! (VLM 판정)"

if cnn_label == "trash":            # VLM 불가 → CNN 폴백
    return "trash", "CNN이 일반쓰레기로 분류"
if cnn_conf < 0.70:                 # 확신도 낮음 → 보수적으로 일반쓰레기 안내
    return "trash", "확신도가 낮아 일반쓰레기로 안내"
return cnn_label, "재질 판별 완료! (CNN 판정)"
```

## 4. CNN 전이학습 (EfficientNet-B0)

### 가. 모델 구조

ImageNet 사전학습 **EfficientNet-B0**의 분류 헤드만 5클래스로 교체하는 전이학습 방식을 사용했다.

```
입력: (3, 224, 224) — YOLO crop 이미지

EfficientNet-B0 백본 (ImageNet 사전학습 가중치)
     ↓
classifier[1]: Linear(1280 → 5)   # 분류 헤드만 교체

출력: 5개 클래스 확률 (plastic / can / glass / paper / trash)
```

### 나. 데이터 증강

실제 촬영 사진은 TrashNet(흰 배경 스튜디오 촬영)과 배경·조명·각도가 달라, 실사진 강건성을 높이는 방향으로 증강을 구성했다.

| 증강 기법 | 파라미터 | 의도 |
|---|---|---|
| RandomResizedCrop | scale 0.6~1.0, ratio 0.8~1.25 | 촬영 거리·구도 변화 시뮬레이션 |
| RandomHorizontalFlip | p=0.5 | 좌우 반전 |
| RandomRotation | ±25° | 기울어진 촬영 각도 |
| ColorJitter | brightness/contrast/saturation 0.35, hue 0.05 | 다양한 조명·색감 |
| RandomPerspective | distortion 0.2, p=0.3 | 원근 왜곡 |
| RandomErasing | p=0.25, scale 0.02~0.15 | 오염·가림(occlusion) 시뮬레이션 |

### 다. 하이퍼파라미터

| 파라미터 | 값 | 설명 |
|---|---|---|
| 백본 | EfficientNet-B0 (IMAGENET1K_V1) | 경량 모델 — CPU 추론 고려 |
| 입력 크기 | 224×224 | EfficientNet-B0 표준 입력 |
| 배치 크기 | 32 | |
| 옵티마이저 | AdamW | |
| 학습률 (lr) | 0.0001 | |
| 스케줄러 | CosineAnnealingLR (T_max=20) | 학습 후반 학습률 점감 |
| 손실함수 | CrossEntropyLoss + 클래스 가중치 | 클래스 불균형 보정 |
| 에폭 | 20 (초기 학습 15) | 증강 강화 후 수렴 여유 확보 |
| 체크포인트 | best val accuracy 시 Drive 저장 | Colab 세션 끊김 대비 |

### 라. 학습 환경

| 항목 | 내용 |
|---|---|
| 플랫폼 | Google Colab |

## 5. VLM 최종 판정 (재질 + 이물질)

### 가. 도입 배경

CNN은 TrashNet(흰 배경 스튜디오 사진)으로 학습되어, 실제 촬영 사진에서 재질을 혼동하는 사례가 반복됐다 — 대표적으로 **기름때 묻은 플라스틱 그릇을 캔으로 오분류**. 증강을 강화해 재학습해도 검증셋 기준 개선이 뚜렷하지 않아(도메인 차이가 원인), 실사진 일반화 능력이 뛰어난 범용 **비전-언어 모델(VLM)을 최종 판단자로 승격**했다.

VLM은 재질 판별과 동시에 **이물질(음식물·기름때·흙) 부착 여부**도 판단해, 오염된 재활용품을 일반쓰레기로 안내하는 역할까지 수행한다.

### 나. 프롬프트 설계

```
이 사진 속 쓰레기 재질을 분류해줘. 다음 중 하나만 골라:
plastic(플라스틱), can(캔/금속), glass(유리), paper(종이/골판지),
trash(음식물 찌꺼기·기름때 등 이물질이 심해 재활용이 불가능한 상태).
이물질이 묻어 있는지도 판단해서,
반드시 '재질,이물질여부' 형식으로 한 줄만 답해 (yes 또는 no).
예시: plastic,yes
```

응답을 `재질,이물질여부` 한 줄로 강제해 파싱을 단순화했고, 파싱 실패 시 `material=None`으로 CNN 폴백된다.

### 다. 로컬/배포 이중화 (`LLM_BACKEND`)

| 환경 | 백엔드 | 모델 | 특징 |
|---|---|---|---|
| 로컬 데모 | Ollama (기본값) | gemma4:e2b (vision) | 무료·무제한, API 키 불필요 |
| 클라우드 배포 | HF Inference API | Llama-3.2-11B-Vision → Qwen2.5-VL-72B → Qwen2-VL-7B 순차 폴백 | 계정 provider에 따라 모델이 막힐 수 있어 다중 폴백 |

- 실패 사유는 조용히 삼키지 않고 `raw` 필드에 담아 앱의 "🔧 VLM 디버그 정보"에서 바로 확인 가능하게 했다.
- HF 크레딧 소진 등으로 전면 실패해도 CNN 판정으로 폴백되어 앱은 항상 동작한다.

## 6. LangChain RAG 챗봇

### 가. 인덱싱 파이프라인 (`rag/ingest.py`)

```
환경부 가이드라인 PDF + 생활법령 md (잡화 부록 8페이지 제외 → 29문서)
     ↓
RecursiveCharacterTextSplitter (chunk 500자, overlap 50자) → 54청크
     ↓
다국어 임베딩 (paraphrase-multilingual-MiniLM-L12-v2, ~470MB 경량)
     ↓
Chroma 벡터DB (rag/chroma_db/) — 배포 첫 부팅 시 자동 구축
```

- 임베딩 모델은 Streamlit Cloud 메모리 제한을 고려해 경량 다국어 모델을 선택했다 (bge-m3는 ~2GB로 과중).

### 나. 검색·답변 생성 (`rag/chain.py`)

```
판별된 재질(한국어) + 사용자 질문
     ↓
"{재질} 분리배출 방법 {질문}" 쿼리로 벡터 검색 (k=8)
     ↓
프롬프트: 참고 문서 + 감지된 재질 + 질문
  "문서에 없는 내용은 지어내지 말고 모른다고 답하세요"
     ↓
LLM 답변 생성 (로컬 Ollama qwen2.5:7b / 배포 HF Qwen2.5-7B-Instruct)
```

### 다. 챗봇 UX

- 업로드 영역 **우측에 상시 노출**되는 카카오톡 스타일 말풍선 패널
- 판별 완료 시 "이 재질의 기본 분리배출 방법"을 자동으로 먼저 답변
- 새 이미지 업로드 시 대화 자동 초기화
- LLM 호출 실패 시 `safe_ask()`가 안내 메시지로 대체 (앱 다운 방지)

## 7. 학습 성과 및 결과 시각화

### 가. 전체 검증 정확도 **93.66%** (EfficientNet-B0 · Colab T4)

#### ○ 학습 곡선

![학습 곡선](training_curve.png)

Train Loss는 꾸준히 감소하고, Val Accuracy는 8 epoch 근처부터 93% 안팎으로 수렴했다.

#### ○ 혼동행렬

![혼동행렬](confusion_matrix.png)

paper·glass·can은 대각선에 집중되어 안정적으로 분류되지만, **plastic↔can 오분류가 일부 잔존**한다 (아래 "다" 참고).

#### ○ 클래스별 Precision / Recall / F1

![클래스별 지표](class_metrics.png)

paper·glass·can은 90%대, trash는 데이터가 가장 적은 클래스(137장)임에도 클래스 가중치 적용으로 F1 80%대를 확보했다.

### 나. 증강 강화 재학습 실험

plastic↔can 혼동을 줄이기 위해 증강을 강화(RandomResizedCrop·강한 ColorJitter·RandomErasing 등)하고 에폭을 15→20으로 늘려 재학습했으나, **검증 정확도는 동일 수준**이었다.

### 다. 한계 분석과 대응

- 원인: TrashNet(흰 배경 스튜디오 사진)과 실제 촬영 사진의 **도메인 차이** — 증강만으로는 극복에 한계
- 대응: 검증셋 성능을 더 쥐어짜는 대신, **실사진 일반화가 강한 VLM을 최종 판단자로 승격**하는 아키텍처 차원의 해결을 선택 (5절 참고)

| # | 이슈 | 원인 → 해결 |
|---|---|---|
| 1 | Streamlit Cloud `cv2` import 실패 | GUI용 opencv가 서버에서 `libGL.so.1` 부재로 실패 → `opencv-python-headless` + `packages.txt(libgl1)` |
| 2 | YOLO 게이트 오판 | COCO 80클래스에 캔·종이가 없어 0개/여러 개 오판 → 0개는 전체 이미지 폴백, 여러 개는 최대 박스 자동 선택 |
| 3 | HF Provider 제약·크레딧 소진 | 계정별 지원 모델 상이(model_not_supported)·402 오류 → 다중 모델 순차 폴백 + 실패 사유 화면 노출 + CNN 폴백 |
| 4 | RAG가 플라스틱/캔 질의에 무관한 문서만 반환 | PDF 잡화 부록의 반복 키워드가 임베딩 유사도 왜곡 (관련 청크가 82개 중 77위) → 해당 페이지 인덱싱 제외 |
| 5 | 로컬에서 VLM이 항상 실패 | VLM이 HF API 전용 구현 → `LLM_BACKEND` 스위치로 로컬 Ollama 이중화 |

## 10. 결론 및 고찰

### 가. 결과 요약

5일 일정 내에 **사진 업로드 → 재질 판별 → 분리수거함 안내 → 배출 방법 질의응답**까지 이어지는 엔드투엔드 파이프라인을 완성하고 Streamlit Cloud에 배포하였다.

- CNN 전이학습(EfficientNet-B0)으로 검증 정확도 **93.66%** 달성 — 클래스 불균형(7배)은 클래스 가중치로 보정
- CNN의 실사진 한계(plastic↔can 혼동)를 데이터가 아닌 **아키텍처(VLM 최종 판단자 승격)로 해결** — 재질 보정과 이물질 검출을 동시에 수행
- 모든 단계에 폴백을 설계해, 외부 API 장애·크레딧 소진 상황에서도 앱이 항상 동작
- 로컬(Ollama)/배포(HF API)를 환경변수 하나로 전환하는 이중화로 데모·배포 환경 모두 지원
